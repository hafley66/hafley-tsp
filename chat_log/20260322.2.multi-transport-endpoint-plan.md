# Session: multi-transport-endpoint-plan

## Goal
Generic `Endpoint` component with pluggable transport bindings. Single .rs file per endpoint: N auto zones (one per transport) + 1 manual zone (transport-agnostic impl). All bindings call the same function. Resolved types (UserSession, DbPool) carry their own per-transport extraction logic, emitted once on the type's file.

## Key design insight
No separate neutral IR. TypeSpec IS the neutral representation. The emitter reads TypeSpec AST + decorators directly. Parameters without a source decorator (@path, @query, @body) are "resolved types" -- the type itself carries extraction logic per transport (e.g. axum FromRequestParts impl). No middleware layer; the type IS the computation.

## Param classification
- **Sourced params**: have @path, @query, @body, @header. Each binding maps source to its extraction syntax.
- **Resolved params**: no source decorator, type is a model (UserSession, DbPool). The type gets a one-time extractor impl per transport emitted on its own file. Bindings emit the type bare (axum) or with explicit extraction (ws).

pub struct UserSession {
    pub user_id: i64,
    pub role: String,

// alloy-axum-extractor-start
#[axum::async_trait]
impl<S> axum::extract::FromRequestParts<S> for UserSession
where S: Send + Sync {
    type Rejection = axum::http::StatusCode;
    async fn from_request_parts(parts: &mut Parts, _state: &S) -> Result<Self, Self::Rejection> {
        parts.extensions.get::<UserSession>().cloned().ok_or(StatusCode::UNAUTHORIZED)

## Design decisions (locked)

- No neutral IR layer; read TypeSpec AST directly
- Manual fn signature: emitted on first generation, frozen. Compiler catches staleness.
- Single file per endpoint (ReplaceFile + N auto zones)
- TransportBinding is a plain data record returned by a factory function
- Resolved types (no source decorator) emit their own per-transport extractor impls
- Orphaned zones: strip on re-emit
- WS binding validates abstraction shape, doesn't need to compile yet
