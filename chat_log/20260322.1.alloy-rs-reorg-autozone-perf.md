# Session: alloy-rs-reorg-autozone-perf

## Goal
Reorganize @hafley/alloy-rs component structure, add Rust ReplaceFile, improve test performance.

## Current State
All changes applied and tested. 15 test files, 143 tests (141 pass, 2 cargo-check skipped by default).

## What Changed

### Package rename
- `hafley-alloy` -> `@hafley/alloy-rs` in package.json

### Component folder reorganization
Flat numbered files -> numbered folders with per-folder numbering:
```
0_primitives/    Generics, Attributes, Serde, Comment, ReferenceTypes
1_declarations/  Struct, Enum, Trait, TypeAlias, Function, Variable, ImplBlock
2_references/    Reference, UseStatement, ModStatements
3_files/         SourceFile, CrateDirectory, ModDirectory
4_codegen/       AppendZone, CodegenPair, Endpoint, ReplaceFile
5_tests/         integration, cargo-check, axum-routing, colocated-routing, auto-manual-split, endpoint-component
```
All internal imports updated (cross-folder siblings, scope depth changes, index.ts, emitter files).

### ReplaceFile for Rust (NEW)
`4_codegen/3_ReplaceFile.tsx` -- same sigil-based replace-region pattern as TsReplaceFile but wrapping Rust's SourceFile (RustSourceFileScope). Exports: ReplaceFile, AutoZone, ManualZone.

### Test performance
- Excluded `5_tests/` (cargo-shelling integration tests) from default vitest run
- Excluded `node_modules/` from test discovery
- Dropped `resolve.conditions: ["source"]` -- stops re-transpiling @alloy-js/core from source every run
- Cargo check tests gated behind `CARGO_CHECK=1` env var via `it.skipIf(!process.env.CARGO_CHECK)`
- Added `test:cargo` script to package.json
- Result: default `pnpm test` runs in ~1.4s (was 24s+)
