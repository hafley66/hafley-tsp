# Session: transport-binding-types

## Goal
Build the type system and extraction matrices for multi-transport endpoint generation. TypeSpec operations emit to N transports (HTTP, WS, CLI) via exhaustive binding factories.

## What was built

### 5_TransportBinding.tsx
- `ClassifiedParam = SourcedParam | ResolvedParam` discriminated union
- `OperationInfo` -- transport-agnostic input to binding factories
- `TransportBinding` -- what a factory produces (Children, not strings)
- Exhaustive extraction matrices: `SOURCED_EXTRACTORS` and `RESOLVED_EXTRACTORS`
  - `Record<TransportId, Record<ParamSource, ExtractorFn>>` enforced via `satisfies`
  - Adding a transport or source without filling all cells = compile error
- `httpBinding()` and `wsBinding()` factories returning JSX Children with Reference components
- WS handler uses typed message struct (e.g. `JoinRoomMsg`) not generic `WsMessage`

### 6_Endpoint.tsx
- Generic component replacing AxumEndpoint
- Takes `ClassifiedParam[]` + `transports: ["http", "ws"]`
- Produces N auto zones (one per transport) + 1 manual zone via ReplaceFile
- All transport bindings delegate to the same manual impl function
- Route registrations teleport to per-transport zones via AppendTo

### 6 snapshot tests passing
1. HTTP-only no params
2. HTTP-only with sourced params (@path, @query)
3. HTTP + WS dual binding
4. PerRequest param (Extension<T> in HTTP, conn.extensions() in WS)
5. Shared param (State<T> in HTTP, conn.state() in WS)
6. Router collection per transport

## Design decisions (locked)

### TypeSpec generics for DI scope, not decorators
- `PerRequest<T>` -- per-request state (axum: Extension<T>, middleware puts it in extensions)
- `Shared<T>` -- process singleton (axum: State<T>, passed via .with_state())
- No custom trait impls generated. Axum built-in extractors handle everything.
- The model (UserSession, DbPool) stays clean. Wrapper declares scope.

### HTTP and WS are both streams
- HTTP is stream(1), WS is stream(N)
- Paginated search is cursor stream: input/join is first event, first page is first response
- No hard modeling wall between transports
- Transport-specific logic (broadcast, lifecycle) is explicit escape hatch

### Path = domain frame, not transport
- Filesystem route tree is domain-driven
- Same path serves HTTP and WS operations
- `@get read(...)` and `@ws join(...)` on the same path/file
- Transport is a property of the operation, not the path

### CLI transport maps from HTTP decorators (no CLI-specific decos needed)
- URL path structure IS the subcommand tree: `/rooms/{room_id}/posts` → `myapp rooms <room_id> posts`
- @path → positional args (URL order)
- @query → --long-flag (no short flags by default)
- @body → final positional or stdin
- @header → env var
- PerRequest types: @env("VAR_NAME") decorator for CLI escape hatch
- Default path needs zero CLI decorators

### Exhaustive satisfies Record pattern
- `TRANSPORT_IDS = ["http", "ws"] as const`
- Adding `"cli"` forces filling all extraction matrix cells
- TypeScript compiler enforces completeness at the emitter level

### Denormalized discriminants
- No stringly-typed dispatch
- TypeSpec decorators (@path, @query) are already denormalized
- Emitter produces typed variants, not strings checked with ===

### Binding factories return JSX Children
- Alloy components (Reference, etc.) used throughout
- Cross-file import resolution via refkeys
- Not string concatenation
