# Session: hafley-alloy-visibility-serde-codegen-pair

## Goal
Add VisibilityContext, serde convenience layer, append zones, and the auto/manual codegen pair pattern to hafley-alloy Rust language package.

## Current State
- Repo: `~/projects/hafley-alloy/` on `master`
- 126 tests across 18 test files (93 fast, 33 cargo-check-gated)
- 4 cargo check validations (original emitter, axum routing, colocated layout, endpoint component)
- Depends on `@alloy-js/core@0.23.0-dev.12`

## What Was Built

### VisibilityContext
- `createContext<string | undefined>(undefined)` -- no default visibility
- `useVisibility(pubProp)`: true→"pub ", false→"" (explicit opt-out), undefined→defer to context
- Emitter wraps tree in `VisibilityContext.Provider value="pub"` so all codegen declarations get pub without prop spam
- Wired into StructDeclaration, EnumDeclaration, FunctionDeclaration, TupleStructDeclaration, StructField

### Serde Layer (no rename)
- `0b_Serde.ts`: `SerdeContainerConfig` and `SerdeFieldConfig` types
- Container: tag, content, untagged, denyUnknownFields, default, transparent
- Field: skip, skipSerializing, skipDeserializing, skipSerializingIf, default, flatten, with, alias
- NO rename/renameAll -- names pass through verbatim
- `serde` prop on StructDeclaration, EnumDeclaration, StructField, TupleStructDeclaration
- Generates `#[serde(...)]` attr strings fed into existing Attributes pipeline

### AppendZone (teleport pattern)
- `13_AppendZone.tsx`: ZoneProvider + Zone + AppendTo
- Reactive arrays via shallowReactive + computed + memo (same pattern as RustModuleScope)
- Child components teleport content into named zones
- Used for: router route collection, dispatch match arms

### CodegenPair (auto/manual file split) -- GENERIC
- `13b_CodegenPair.tsx`: CodegenPair, AutoFile, StubFile, ImplCall
- `_auto.rs` files: regenerated, protocol boilerplate, delegates to impl
- `.rs` files: stubbed once, user-owned implementation, pure domain logic
- ImplCall renders delegation path from ImplPathContext
- Protocol-agnostic -- HTTP Endpoint is just one skin on top

### Endpoint (HTTP/axum skin on CodegenPair)
- `13a_Endpoint.tsx`: uses CodegenPair internally
- Generates paired auto+stub files, axum extractors, Json wrapping
- AppendTo for router zone route collection

### Colocated Model Placement
- Models live as deep as possible, hoist to common ancestor when cross-referenced
- No separate models/ folder needed
- Refkey system handles use crate:: resolution regardless of placement

## Key Insights
- VisibilityContext pattern: context defaults differ between component tests (no pub) and codegen (pub everything). Context layering solves this without breaking existing tests.
- Serde rename is evil in codegen -- names pass through verbatim from input schema
- AppendZone uses identical reactive pattern to RustModuleScope (shallowReactive Set → shallowReactive Array)
- CodegenPair is protocol-agnostic: same primitive works for HTTP, WebSocket, gRPC, CLI, events
- Rust module collision: can't have foo.rs + foo/mod.rs. Use distinct names (list_users.rs + list_users_auto.rs)
- Colocated models with hoisting > lateral technical folder separation (models/ + routes/)
- The auto file's delegation call needs crate:: prefix since modules can't resolve siblings without it
